# スレッドとエラー

## API を呼ぶスレッド

`IAudioEngine` は、複数のスレッドから同時に使えません。`Initialize`、`Play`、`State`、`SetVolume`、`Stop`、`DestroyPlayback`、`StopAll`、`Shutdown` は、すべて同じ音声処理用スレッドから呼びます。

XAudio2 と OpenAL のバックエンドは、音声処理用スレッドで COM と Media Foundation を初期化し、同じスレッドで終了します。エンジンを別のスレッドへ移したり、音声デバイスのコールバックから直接操作したりしないでください。

aEngine 内で複数の OpenAL エンジンを使う場合、OpenAL コンテキストの切り替えは、プロセス全体で1つずつ順番に行われます。各 API の処理が終わると、呼び出し前に使われていた外部のコンテキストへ戻します。

aEngine の API を実行している間は、別のスレッドにある外部の OpenAL コードから、標準の `alcMakeContextCurrent` や OpenAL リソースの操作を行わないでください。外部の OpenAL コードと同時に使う場合は、利用側ですべてのコンテキスト操作を1つずつ実行するか、外部側で `ALC_EXT_thread_local_context` を独立して使える必要があります。

バックエンド内部の音声デバイス用コールバックは、再生が終わったことを内部へ知らせるためだけに使います。このコールバックから利用側のコールバックを直接実行することはありません。再生状態は、音声処理用スレッドから `State` で確認してください。

## エラー

失敗する可能性がある API は、`Expected<T>` または `Result` を返します。主なエラーコードは次のとおりです。

- `InvalidArgument`: 音源や音量の値が不正、または同じバックエンドを重複して登録した
- `Unsupported`: 選んだバックエンドが入力形式や操作に対応していない。バージョン 0.1 では、OpenAL へモノラルとステレオ以外の PCM を渡した場合などに返る
- `BackendUnavailable`: バックエンドが未登録、または必要な実行用 DLL を読み込めない
- `DeviceUnavailable`: 既定の音声デバイスがない、使用中、または無効になっている
- `InitializationFailed`: COM、Media Foundation、OS や外部ライブラリのエンジンやコンテキストなど、音声デバイス以外の初期化に失敗した
- `ResourceLoadFailed`: ファイルを開けない
- `DecodeFailed`: 圧縮された音声をデコードできない
- `PlaybackFailed`: OS や外部ライブラリが持つ再生機能の操作に失敗した
- `InvalidHandle`: 無効なハンドル、または更新番号が古いハンドルを渡した
- `IoError`: 一般的な入出力の失敗に備えて予約している。バージョン 0.1 の組み込みバックエンドは現在返さない

`DiagnosticCallback` は、原因を調べるための補助情報を受け取るものです。API が成功したかどうかは、必ず戻り値で判断してください。このコールバックは、API の処理中に同じ音声処理用スレッドで実行されます。コールバックの中から、同じエンジンの API を呼ばないでください。初期化中や終了中に同じエンジンの API を呼ぶと、`InitializationFailed` になります。コールバックが例外を投げた場合は、エンジンとの境界で捕捉して破棄します。
