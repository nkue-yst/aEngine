# API とデータの管理

## エンジンを作る

利用側で `BackendRegistry`（使えるバックエンドの一覧）を用意し、必要なバックエンドだけを登録します。`BackendRegistry::Create` はバックエンドのオブジェクトを作り、その場で初期化します。通常は、作成後に `Initialize` をもう一度呼ぶ必要はありません。

未登録のバックエンド、同じバックエンドの重複登録、初期化の失敗は `Error` として返ります。`BackendRegistry` は、プログラム全体で自動的に1つだけ共有される仕組み（シングルトン）ではありません。そのため、複数の利用箇所に別の `BackendRegistry` を持てます。設定名を `BackendId` へ変換する方法と、初期化に失敗したときに別のバックエンドへ切り替えるかどうかは、利用側で決めます。

初期化済みのエンジンへ `Initialize` をもう一度呼ぶと、現在の再生をすべて終了し、バックエンドを初期化し直します。それまでのハンドルはすべて無効になります。通常は、使用中のハンドルを明示的に破棄してからエンジンを作り直す方が、データの状態を管理しやすくなります。

## PlaybackHandle

`PlaybackHandle` は、再生を安全に識別するハンドル ID です。内部には、リソース表の位置を示す `index`、古い参照を検出するための64ビットの `version`（更新番号）、作成元のエンジンを示す値が入っています。有効なハンドルを作るのは `Play` だけです。既定のコンストラクターで作ったハンドルは無効です。

`DestroyPlayback` で破棄したハンドル、同じ位置が再利用された後の古いハンドルを使うと、`InvalidHandle` が返ります。更新番号が上限に達した位置は再利用しないため、番号が一周して古いハンドルが再び有効になることはありません。

基本的な使用の流れは次のとおりです。

1. `Play` でハンドルを受け取る。
2. `SetVolume`、`State`、`Stop` で再生を操作する。
3. `Finished` または `Stopped` になったことを確認し、`DestroyPlayback` でハンドルを破棄する。

`Stop` はハンドルを破棄しません。`StopAll` も、すべてのハンドルを `Stopped` 状態にするだけです。利用側で、保持しているハンドルを `DestroyPlayback` してください。エンジンの `Shutdown` とデストラクターは、残っている再生をすべて破棄し、それまでのハンドルを無効にします。

### 独自バックエンドを実装する場合

`IAudioEngine` を直接実装するバックエンドは、`protected` の `MakePlaybackHandle(index, version)` でハンドルを作ります。`version` は1から始め、同じ位置を再利用するたびに増やしてください。`version` の `0` は常に無効です。操作を受け付ける前に `OwnsPlaybackHandle(handle)` を確認し、その後で `index` と `version` をバックエンド自身の管理表と照合します。

## 音量

音量には、`0.0f` から `1.0f` までの有限の値を指定します。範囲外の値、NaN、Infinity を指定すると `InvalidArgument` になります。範囲内への自動補正は行いません。

## State

- `Playing`: バックエンドが再生を開始しており、終了も停止もしていない。
- `Stopped`: 利用側が停止した。
- `Finished`: 音源末尾まで自然再生した。

最後まで再生したハンドルも、`DestroyPlayback` を呼ぶまでは有効です。

## バックエンドが対応する機能

`BackendCapabilities` は、基本的な再生、再生ごとの音量変更、3D 音声、エフェクト、ストリーミング再生に対応しているかを表します。バージョン 0.1 の実際に音を出すバックエンドは、基本的な再生と、再生ごとの音量変更に対応します。

利用側でのデータの持ち方と、音源の読み出し方法については、[組み込みガイド](INTEGRATION.md)を参照してください。
